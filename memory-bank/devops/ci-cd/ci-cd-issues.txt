Проблемы с CI/CD пайплайном
1. Проблема с миграцией - команда kubectl exec deploy/gprod-backend при первом деплое может не выполниться
2. Несоответствие namespace - в deploy.yml используется namespace gprod, хотя в документации упоминается develop-gprod
3. Проблема с откатом - используется команда helm rollback к ревизии 1, а не к предыдущей версии
4. Использование тега latest вместо конкретных версий или SHA коммита
5. Отсутствие кеширования зависимостей между запусками пайплайна
6. Отсутствие настройки Node.js версии в шагах lint и test
7. Отсутствие проверки уязвимостей образа контейнера
8. Отсутствие управления переменными окружения для разных стадий (dev/stage/prod)
9. Различия между deploy.yml и deploy.yml.bak в уведомлениях о периодическом healthcheck
10. Отсутствие стратегии деплоя (Rolling Update, Recreate) в конфигурации Helm
